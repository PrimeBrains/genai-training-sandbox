# 事前セットアップ案内（研修当日までに必ず完了してください）

対象: Windows PC（コマンドはすべて **PowerShell** で実行します）
所要時間の目安: 30分（ほとんどはダウンロード待ちです）

## 1. インストールするもの

| ツール | 入手先 | 確認コマンド（PowerShell） |
|--------|--------|---------------------------|
| JDK 21 | https://adoptium.net/ （Temurin 21 の **.msi**） | `java -version` |
| Git for Windows | https://git-scm.com/ | `git --version` |
| GitHub CLI | https://cli.github.com/ | `gh --version` |
| Claude Code | https://claude.com/claude-code | `claude --version` |

- JDK のインストーラでは **「Add to PATH」（環境変数への追加）を有効**にしてください（Windows で一番多い詰まりどころです）
- Git for Windows は既定設定のままでOK。Git Bash を開く必要はありませんが、Claude Code が内部で使うため**インストールは必須**です
- インストール後は **PowerShell を開き直してから**確認コマンドを実行してください（PATH が反映されないため）

## 2. GitHub の認証

```powershell
gh auth login
```

ブラウザが開くので、自分の GitHub アカウントでログインしてください。
（アカウントがない方は https://github.com/ で無料登録してから）

## 3. AI 利用キーの設定

研修運営から `genai-training-pairXX-credentials.txt` というファイルがペアごとに配布されます。
PowerShell で次を実行してファイルを開き:

```powershell
mkdir $env:USERPROFILE\.aws -Force
New-Item $env:USERPROFILE\.aws\credentials -ItemType File -Force
notepad $env:USERPROFILE\.aws\credentials
```

（`New-Item` を挟まず `notepad` だけで開くと、メモ帳が勝手に拡張子を補って `credentials.txt` を作ってしまい、設定が読み込まれません）
開いたメモ帳に、配布ファイルの3行を貼り付けて保存してください:

```
[genai-training]
aws_access_key_id = （配布された値）
aws_secret_access_key = （配布された値）
```

- これだけで Claude Code の接続設定は完了です（残りの設定はリポジトリに同梱済み）
- ⚠ このキーは研修専用・研修後に無効化されます。**他人に共有したり、リポジトリにコミットしたりしないでください**

## 4. 動作確認（ここまで到達したら完了）

ペア用リポジトリ（運営から URL を連絡します）を clone してテストを実行:

```powershell
git clone https://github.com/PrimeBrains/genai-training-sandbox-pairXX.git
cd genai-training-sandbox-pairXX
.\gradlew.bat test
```

- 初回は依存ライブラリのダウンロードで数分かかります（研修当日の時間節約のため、必ず事前に1回実行しておいてください）
- **`BUILD FAILED`（テスト失敗）と表示されたら成功です。** テストが落ちているのは仕様で、研修当日の課題です。直さないでそのままにしておいてください

最後に Claude Code の接続確認（同じフォルダの中で実行）:

```powershell
claude --print "接続テスト。10文字以内で応答して"
```

応答が返ってくればすべて完了です。

## うまくいかないとき

- 完了/未完了を、前日までに運営へ一言連絡してください（未完了でも当日サポートします）
- エラーメッセージのスクリーンショットを添えてもらえると当日の対応が速くなります
- よくある原因: 確認コマンドが「認識されません」→ PowerShell の開き直し / JDK の「Add to PATH」漏れ
- 接続確認で `Could not load credentials` → `Get-ChildItem $env:USERPROFILE\.aws` でファイル名が `credentials`（`.txt` なし）になっているか確認。`credentials.txt` になっていたら `Rename-Item $env:USERPROFILE\.aws\credentials.txt credentials` で直す
